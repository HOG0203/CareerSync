# PDCA Design: 학생 성적 엑셀 일괄 입력 테스트 시스템

## 1. 개요
본 문서는 `전기3-1.xlsx` 파일을 활용하여 스마트전기과 3학년 1반 학생들의 성적 데이터를 시스템에 일괄 등록하고 검증하는 테스트 페이지의 상세 설계를 기술한다.

## 2. 데이터베이스 설계
개별 과목별 성적을 유연하게 저장하기 위해 기존 `students` 테이블과 연관된 별도의 성적 테이블을 구성한다.

### 2.1 `student_scores` 테이블 (신규)
| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | UUID | PK (Default: uuid_generate_v4()) |
| `student_id` | UUID | FK (students.id) |
| `academic_year` | INTEGER | 학년도 (예: 2026) |
| `grade` | INTEGER | 학년 (1, 2, 3) |
| `semester` | INTEGER | 학기 (1, 2) |
| `subject` | TEXT | 과목명 |
| `score` | NUMERIC | 원점수 (예: 91) |
| `average_score` | NUMERIC | 과목 평균 (예: 80.3) |
| `standard_deviation`| NUMERIC | 표준편차 (예: 9.0) |
| `created_at` | TIMESTAMPTZ| 생성일 |

## 3. UI/UX 설계

### 3.1 테스트 페이지 (`/admin/grade-import-test`)
- **파일 업로드 영역**: `.xlsx` 파일을 선택할 수 있는 Dropzone 또는 File Input.
- **데이터 분석 및 매칭 결과**:
  - 엑셀 내 학생 정보(이름, 번호)와 DB 학생 정보의 매칭 여부 표시.
  - 추출된 과목 및 성적 데이터 미리보기 그리드.
- **실행 버튼**: 'DB에 일괄 저장' 버튼 (매칭 성공 학생들만 대상).

### 3.2 내비게이션 메뉴
- 사이드바(`Nav`) 관리자 섹션에 '성적 일괄 입력(T)' 메뉴 추가.
- 모바일 하단 탭(`MobileBottomTab`)에 임시 버튼 노출 (테스트 기간 한정).

## 4. 로직 설계

### 4.1 Excel 파싱 (Client-side)
- `xlsx` 라이브러리를 사용하여 파일을 바이너리로 읽어들임.
- 행(Row) 단위로 순회하며 학생 번호/이름이 바뀔 때까지 과목 정보를 수집.
- `원점수/평균(편차)` 형식의 문자열(예: `91/80.3(9.0)`)을 정규표현식으로 파싱하여 개별 수치로 분리.

### 4.2 매칭 및 저장 (Server Action)
- `student_name`과 `student_number`를 키로 사용하여 현재 2026학년도 3학년 1반 학생의 UUID를 조회.
- `upsert` 로직을 사용하여 동일 학년도/학기/과목의 데이터가 이미 존재할 경우 업데이트 처리.

## 5. 검증 계획
1. `전기3-1.xlsx` 파일을 업로드하여 177개 레코드(또는 실제 인원수)가 정상 파싱되는지 확인.
2. 매칭되지 않는 학생(이름 오타 등)에 대한 예외 처리 메시지 확인.
3. 소수 학생의 데이터를 DB에 반영 후 대시보드 또는 학생 상세에서 성적이 올바르게 조회되는지 검증.
