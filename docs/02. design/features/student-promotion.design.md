# PDCA Design: 학년별 분리 관리 및 진급 시스템

## 1. 페이지별 인터페이스 설계

### 1.1 학생 취업 및 현장실습 현황 (`/students`)
- **대상**: `graduation_year == baseYear` (3학년) 전용.
- **용도**: 전체 3학년 데이터의 수평적 관리 및 취업 통계 확인.
- **제한**: 1, 2학년 데이터는 표시하지 않음.

### 1.2 학반관리 (`/class-management`)
- **대상**: 담임교사가 담당하는 특정 반의 **1, 2, 3학년 전체**.
- **학년 선택**: 학년(1/2/3) 탭 또는 필터를 통해 해당 반의 학년별 학생 조회.
- **상담 중심 컬럼**: 
  - `counseling_notes`: 담임교사 상담 일지 (개별 관리 데이터).
  - `career_path`: 희망 진로 방향 (저학년 기초 진로 조사).
  - `target_certificates`: 취득 목표 자격증.

## 2. 데이터베이스 및 API 설계

### 2.1 스키마 확장
- `student_employments` 테이블에 저학년 상담용 필드 추가:
  - `counseling_log` (text): 개별 상담 기록.
  - `career_aspiration` (text): 기초 진로 희망.

### 2.2 학반관리 필터링 로직
- `grade`: `baseYear`와의 차이로 자동 계산.
- `major`, `class_info`: 해당 교사가 담당하는 반 데이터만 조회.

## 3. 진급(Promotion) 로직 상세
- 진급 버튼 클릭 시 해당 학급 학생들의 `graduation_year`를 -1 처리.
- 1학년(2028년 졸업) → 2학년(2027년 졸업)으로 정보 일괄 업데이트.
- 기존의 상담 기록(`counseling_log`)은 그대로 유지되어 누적 관리 가능.
