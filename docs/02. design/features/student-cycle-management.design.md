# PDCA Design: 학년도 전환 및 학생 데이터 관리 고도화

## 1. 개요
- **배경**: 매년 새로운 3학년 학생들의 취업 데이터를 관리해야 함.
- **목표**: 기존 학생(2026년 졸업)과 신규 학생(2027년 졸업 등)을 구분하여 관리하고, 대량 등록 시 연도 정보를 포함함.

## 2. 세부 설계 사항

### 2.1 CSV 업로드 양식 업데이트 (`import-button.tsx`)
- **변경 사항**: 템플릿 헤더에 `졸업연도` 필드 추가.
- **순서**: `순번,졸업연도,학과,학반,번호...`
- **기대 효과**: 파일 업로드 시점에 학생의 기수(졸업연도)를 명확히 지정 가능.

### 2.2 서버 액션 수정 (`src/app/students/actions.ts`)
- **변경 사항**: `uploadStudentsCSV` 함수에서 CSV의 두 번째 컬럼(`졸업연도`)을 파싱하여 DB의 `graduation_year` 컬럼에 매핑.
- **유효성 검사**: 졸업연도가 숫자인지 확인하고, 값이 없을 경우 현재 연도 혹은 기본값 적용.

### 2.3 데이터 조회 및 필터링 강화
- **학생 페이지**: 상단 필터 바 혹은 헤더 필터에 `졸업연도` 필터를 기본으로 제공하여, 이전 졸업생 데이터를 손쉽게 조회하거나 숨길 수 있도록 함.
- **대시보드**: 상단 '연도' 선택 시 해당 연도의 취업률 및 통계가 즉시 반영되도록 데이터 페칭 로직 최적화.

## 3. 학년도 전환 시나리오 (Operation Manual)
1. **졸업 처리**: 학년말이 되면 모든 학생의 취업 현황을 최종 확인.
2. **데이터 아카이빙**: 시스템 설정에서 특정 졸업 연도의 편집 권한을 비활성화(Read-only) 처리 (선택 사항).
3. **신규 등록**: 다음 해 3학년 학생들의 정보를 담은 CSV 파일을 '가져오기'를 통해 업로드 (졸업연도: 2027 지정).
4. **결과 확인**: 대시보드에서 2027년도 졸업생 데이터가 정상적으로 집계되는지 확인.

## 4. 데이터베이스 제약 조건 확인
- `student_id`가 중복되지 않도록 주의 (학번 기반인 경우 연도와 조합하거나 고유 번호 사용 권장).
