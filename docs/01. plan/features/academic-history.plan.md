# PDCA Plan: 전 학년 학적 이력 영구 기록 시스템

## 1. 개요 (Overview)
- **목표**: 학생이 진급하거나 소속이 변경되더라도 1, 2, 3학년 당시의 학과, 반, 번호 정보를 모두 데이터베이스에 보존하고 조회할 수 있는 체계 구축.
- **주요 기능**:
  - **학적 이력 테이블**: 학년도별 소속 정보를 저장하는 전용 테이블 운영.
  - **자동 기록 인계**: 진급 처리 시 현재 정보를 이력으로 자동 저장.
  - **통합 이력 뷰어**: 상담일지 및 학생 상세 정보에서 과거 소속 이력 확인.

## 2. 분석 및 설계 (Analysis & Design)

### 2.1 데이터 모델링
- **신규 테이블**: `student_academic_history`
  - `student_id`: UUID (FK, `student_employments.id`)
  - `grade`: INTEGER (1, 2, 3)
  - `academic_year`: INTEGER (연도)
  - `major`, `class_info`, `student_number`: 해당 시점의 정보

### 2.2 비즈니스 로직
- **저장 시점**: 
  1. 학생 신규 등록 시 현재 학년 정보 저장.
  2. `promoteStudents` 액션 실행 시 진급 후의 정보를 신규 이력으로 추가.
- **조회**: 학생 고유 ID(UUID)를 기준으로 전체 이력을 정렬하여 출력.

## 3. 단계별 추진 계획 (Step-by-Step Plan)

### Phase 1: DB 스키마 구축
- [ ] `student_academic_history` 테이블 생성 및 관계 설정.
- [ ] 기존 등록된 학생들의 현재 정보를 이력 테이블의 초기 데이터로 마이그레이션.

### Phase 2: 서버 액션 및 연동 로직 수정
- [ ] `promoteStudents` 함수에서 업데이트와 동시에 이력 테이블 `insert` 로직 추가.
- [ ] 학생 개별 수정 및 일괄 업로드(Import) 시에도 이력 동기화 적용.

### Phase 3: UI 구현
- [ ] `CounselingModal` 상단에 학적 이력 표시 컴포넌트 추가.
- [ ] (옵션) 학생 관리 테이블에서 과거 이력을 툴팁 등으로 확인하는 기능.

## 4. 기대 결과 (Expected Outcome)
- 학생이 졸업한 후에도 3년간의 소속 변화를 정확하게 추적 가능.
- 과거 데이터와의 대조를 통해 학급 편성 및 진로 지도 통계의 정확성 향상.
