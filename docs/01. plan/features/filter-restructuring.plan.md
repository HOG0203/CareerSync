# PDCA Plan: 학사학년도 및 학년 필터 분리 체계 구축

## 1. 개요 (Overview)
- **목표**: 사용자 인터페이스(UI)에서 '학사학년도'와 '학년'을 각각 선택할 수 있도록 분리하여, 학교 현장 업무와 일치하는 직관적인 필터링 환경 제공.
- **배경**: 현재는 '졸업연도'가 필터의 기준이 되어 있어, 특정 학년도의 1, 2학년 데이터를 조회하거나 과거 이력을 확인할 때 혼동이 발생함.

## 2. 설계 및 사양 (Design & Spec)

### 2.1 데이터 매핑 로직 (The Formula)
- **핵심 공식**: `GraduationYear = AcademicYear + (4 - Grade)`
- **동작 예시**:
    - 2026학년도 3학년 선택 시 -> 2027년 졸업 예정자 데이터 조회
    - 2026학년도 2학년 선택 시 -> 2028년 졸업 예정자 데이터 조회
    - 2025학년도 3학년 선택 시 -> 2026년 졸업 완료자 데이터 조회

### 2.2 URL 파라미터 명세
- `ay`: 학사학년도 (Academic Year, 예: 2024, 2025, 2026)
- `grade`: 학년 (1, 2, 3)
- *기존 `year` 파라미터는 하위 호환성을 위해 내부적으로 `ay`와 `grade` 조합으로 변환 처리.*

### 2.3 UI 변경 사항
- **DashboardFilters**: 
    - [학년도 선택] (2024, 2025, 2026...)
    - [학년 선택] (1학년, 2학년, 3학년)
    - 두 값이 변경될 때마다 URL을 갱신하고 데이터를 다시 로드함.
