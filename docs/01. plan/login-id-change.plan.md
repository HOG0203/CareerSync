# [Plan] 로그인 아이디를 이름으로 변경

## 1. 개요
현재 이메일 기반인 로그인 방식을 사용자 이름(Username) 기반으로 변경하여 사용자 편의성을 높입니다.

## 2. 목표
- 사용자가 이메일 대신 고유한 '이름' 또는 '아이디'를 입력하여 로그인 가능
- 기존 Supabase Auth 인프라를 유지하면서 로그인 흐름만 개선

## 3. 주요 변경 사항

### 3.1 데이터베이스 (SQL)
- 사용자 테이블에 `username` 컬럼 추가 (Unique 제약 조건)
- 기존 데이터가 있을 경우를 대비한 마이그레이션 전략 수립

### 3.2 백엔드 (Auth Actions)
- `login` 액션 수정: 입력받은 `identifier`가 이름인지 이메일인지 판별하거나, 이름을 기반으로 실제 이메일 조회 후 인증 진행
- `signup` 액션(필요 시) 수정: 가입 시 중복된 이름 검사 로직 추가

### 3.3 프론트엔드 (UI)
- `src/app/login/page.tsx`: 이메일 입력창 라벨 및 Placeholder 변경
- 관련 유효성 검사 로직 업데이트

## 4. 할 일 목록 (Tasks)
- [ ] DB 스키마 업데이트 (username 필드 추가)
- [ ] 로그인 서버 액션 수정
- [ ] 로그인 페이지 UI 수정
- [ ] 테스트 및 검증

## 5. 예상 결과
- 사용자는 본인의 이름(아이디)과 비밀번호만으로 서비스 이용 가능
