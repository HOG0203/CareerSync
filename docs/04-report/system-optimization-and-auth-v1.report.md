## 2. 주요 구현 내용

### 2.1 시스템 설정 스토리지 마이그레이션
* **문제**: 서버리스 배포 환경에서 로컬 파일(`system-settings.json`) 수정 불가.
* **해결**:
  * Supabase 데이터베이스에 `system_settings` 및 `master_certificates` 테이블 생성.
  * 서버 액션(`actions.ts`)을 파일 I/O에서 Supabase DB 기반 읽기/쓰기로 전면 전환.
  * 관리자만 수정 가능하도록 RLS(Row Level Security) 정책 적용 완료.

### 2.2 전방위적 성능 최적화 (성능 2배 이상 개선)
* **대상 페이지**: 대시보드 메인, 취업 상세현황, 학반 관리, 학생 등록 및 진급 관리
* **최적화 기법**:
  * **병렬 데이터 패칭**: `await Promise.all()`을 적용하여 사용자 권한, 시스템 설정, 연도 목록, 학생 기초 데이터를 동시에 요청하도록 개선.
  * **단일 루프 최적화**: 배열 데이터(수천 건)에 대해 중첩된 `filter`와 `map`을 제거하고, 단 한 번의 `for` 루프로 그룹화 및 통계 계산을 완료.
  * **조회 속도 향상**: `Map` 및 `Set` 구조를 적극 활용하여 데이터 매칭 시간을 O(n)에서 O(1) 수준으로 최적화.

### 2.3 계정 보안 및 편의성 강화
* **사용자 프로필 설정**: 우측 상단 드롭다운 메뉴에 '프로필 설정' 모달 추가.
  * 안전한 비밀번호 변경 워크플로우 구현.
  * Radix UI(shadcn/ui)의 고질적인 `pointer-events` 잠금 버그 완벽 대응 (강제 스타일 초기화 로직 및 지연 렌더링).
* **관리자용 비밀번호 초기화**: 사용자 관리 테이블에 계정 잠금/분실 대비용 초기화 버튼 추가.
  * 클릭 시 Supabase Admin API를 통해 즉시 '123123'으로 비밀번호 강제 재설정.
