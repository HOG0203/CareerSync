# [REPORT] 학반 관리 용어 통일 및 동적 진로코스 시스템 구현

## 1. 개요
사용자 피드백을 바탕으로 학반 관리 및 학생 현황 페이지의 주요 용어를 직관적으로 수정하고, 진로 희망 선택에 따라 세부 코스가 자동으로 필터링되는 동적 드롭다운 시스템을 구축한 프로젝트입니다.

## 2. 주요 수정 내용

### 2.1 용어 체계 정립 (Terminology)
- **기초진로희망 → 진로희망**: 모든 UI 및 문서, DB 코멘트에서 용어를 간소화 및 통일했습니다.
- **특이사항 → 진로코스**: 학급 관리의 목적에 맞게 단순 비고가 아닌 교육 과정(도제, 축구부 등)을 의미하는 '진로코스'로 명칭을 변경했습니다.
- **레이블 개선**: 표 가독성을 위해 '진로코스 및 특이사항'과 같이 줄바꿈과 병기 레이블을 적용했습니다.

### 2.2 동적 드롭다운 시스템 (Dynamic Options)
- **엔진 확장**: `StandardSpreadsheetTable` 컴포넌트가 행 데이터(`rowData`)를 기반으로 옵션을 실시간 계산하는 함수형 `options` 설정을 지원하도록 엔진을 업그레이드했습니다.
- **의존성 로직 적용**:
  - **취업** 선택 시: 대/공기업, 공무원, 중견기업 등 취업 관련 코스만 표시.
  - **제외인정자** 선택 시: 군특성화, 기술사관, 운동부 등 특수 관리 대상 표시.
  - **진학** 선택 시: 세부 코스 선택 불필요로 인한 목록 비움 처리.

### 2.3 데이터 일관성
- 대시보드 차트(`CareerAspirationChart`)의 정렬 순서와 범례를 수정된 '진로희망' 3종(취업, 진학, 제외인정자)에 최적화했습니다.

## 3. 향후 권장 사항
- **기존 데이터 마이그레이션**: 용어가 '기초진로희망'에서 '진로희망'으로 축소됨에 따라, 기존에 상세하게 입력된 데이터들을 새로운 3종 카테고리에 맞게 정제하는 작업이 필요할 수 있습니다.
