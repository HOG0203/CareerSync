# 📊 CareerSync 성능 진단 및 UI 업데이트 보고서 (v1.6)

**업데이트 일자:** 2026-03-09  
**작성자:** bkit (Gemini CLI)

## 1. ✅ 최근 업데이트 요약 (UI/UX 개선)

### 1.1 진로희망 옵션 고도화
- **신규 옵션 추가**: 모든 학년의 진로희망 드롭다운에 **'운동부'** 항목을 추가하여 특수 목적 학생 관리를 지원합니다.
- **정렬 순서 최적화**: 1학년의 경우, 최근 학부모 및 학생들의 선호도를 반영하여 **'가업승계'** 옵션을 '진학희망' 보다 상위로 노출되도록 조정했습니다.
- **통계 정규화**: 신규 추가된 '운동부' 옵션이 대시보드 차트(`CareerAspirationChart`)의 범례와 막대 그래프에 정상적으로 집계되도록 로직을 동기화했습니다.

## 2. 🐢 페이지 로딩 성능 진단 (Bottleneck Analysis)

현재 관리자 페이지(학반 관리, 학생 취업 현황)의 로딩이 느린 핵심 원인은 다음과 같습니다.

### 2.1 과도한 데이터 패칭 (Over-fetching)
- **증상**: 특정 학급(약 20~30명)만 조회하더라도 시스템이 해당 학년 전체(수백 명)의 데이터를 DB에서 한 번에 가져와 메모리에서 필터링합니다.
- **영향**: 데이터 전송량(Payload)이 불필요하게 커지며, 네트워크 지연 시간이 증가합니다.

### 2.2 중복된 DB 호출 (Frequent Metadata Queries)
- **증상**: 페이지를 열 때마다 `시스템 설정`, `졸업연도 목록`, `마스터 자격증` 등 변하지 않는 기초 데이터를 매번 새롭게 쿼리합니다.
- **영향**: 서버 사이드 렌더링(SSR) 준비 시간이 길어집니다.

### 2.3 클라이언트 렌더링 부하 (Spreadsheet Overhead)
- **증상**: `StandardSpreadsheetTable`에서 수십 개의 컬럼과 수백 개의 셀이 동시에 렌더링되면서 브라우저 메인 스레드 점유율이 상승합니다.
- **영향**: 스크롤이 끊기거나 필터 전환 시 반응이 느려집니다.

## 3. 🚀 향후 성능 개선 로드맵 (Proposed Fixes)

| :--- | :--- | :--- | :--- |
