# 학생 및 학급 관리 페이지 동기화 및 UI 최적화 보고서 (v1.7)

## 📝 1. 개요
학급 관리(`class-management`) 페이지와 학생 관리(`students`) 페이지 간의 데이터 항목 불일치를 해소하고, 사용자 피드백을 반영하여 UI를 정돈했습니다. 특히 진로코스 항목의 세분화와 불필요한 컬럼 제거를 통해 데이터 관리의 효율성을 높였습니다.

## 🚀 2. 주요 변경 사항

### 2.1 진로코스 및 특이사항 옵션 확장
- **신규 항목 추가**: '제외인정자' 진로희망 선택 시 표시되는 하위 옵션에 **'특수교육대상자'**를 추가했습니다.
- **위치**: '운동부' 항목 바로 아래에 배치하여 특수 목적 학생 관리를 용이하게 했습니다.
- **적용 페이지**: 학급 관리, 학생 관리 페이지 공통 적용.

### 2.2 학생 관리 페이지(`students`) 기능 강화 및 동기화
- **컬럼 이식**: 학급 관리 페이지에만 존재하던 '진로코스 및 특이사항' 컬럼을 학생 관리 페이지에도 동일하게 이식했습니다.
- **동적 옵션 적용**: 진로희망(취업/진학/제외인정자)에 따라 드롭다운 목록이 실시간으로 변하는 로직을 적용했습니다.
- **DB 연동**: `special_notes` 필드를 `BASIC_INFO_FIELDS`에 추가하여 학생 관리 페이지에서도 즉시 수정 및 저장이 가능하도록 개선했습니다.

### 2.3 UI/UX 최적화 및 정돈
- **자격증 컬럼 제거**: 학생 관리 페이지의 메인 테이블에서 '자격증' 컬럼을 제거하여 가로 스크롤을 줄이고 핵심 정보에 집중할 수 있도록 했습니다.
- **헤더 통합**: '기기/자격' 그룹 헤더를 **'비고(특이사항)'**로 변경하고, 하위 컬럼 라벨을 '비고'로 간소화하여 시각적 복잡도를 낮췄습니다.

## 🛠️ 3. 기술적 수정 사항
- **`src/app/students/actions.ts`**: `special_notes` 필드를 기본 정보 필드 목록에 추가.
- **`src/app/(dashboard)/students/student-table.tsx`**: 
  - `GET_CAREER_COURSE_OPTIONS` 함수 구현 및 적용.
  - `COLUMNS` 및 `GROUP_HEADERS` 구조 재설계.
- **`src/app/(dashboard)/class-management/class-table.tsx`**: '특수교육대상자' 옵션 추가.

## ✅ 4. 기대 효과
- 학급 담임 교사와 관리자 간의 데이터 관리 기준 통일.
- 특수교육 대상 학생에 대한 체계적인 트래킹 가능.
- 테이블 UI 간소화를 통한 가독성 및 데이터 입력 편의성 향상.
