# [REPORT] 다학년 통합 대시보드 및 학반 관리 고도화

## 1. 개요
기존 3학년 중심의 대시보드를 전 학년으로 확장하고, 학반 관리 데이터와의 실시간 동기화 및 사용자 편의성(Undo/Redo, 직접 입력 등)을 대폭 개선한 프로젝트입니다.

## 2. 주요 구현 내용

### 2.1 다학년 통합 대시보드
- **학년별 전용 뷰**: 1, 2학년은 진로 탐색 및 준비 지표(진로희망, 자격증, 병역) 중심, 3학년은 취업 결과 중심의 UI 제공.
- **지능형 필터(Smart Default)**: 
  - 관리자/비담임: 3학년 기본값.
  - 담임 교사: 본인 담당 학년(`assigned_grade`) 자동 선택.
- **차트 인터페이스 고도화**:
  - 모든 주요 차트에 '분포(도넛)' vs '학과별/반별(막대)' 토글 기능 추가.
  - 학과 필터 선택 시 자동으로 '반별' 데이터로 전환 및 단위('반') 표시.
  - 모든 도넛 차트 디자인 통일 (9시 시작, 시계 방향, 지시선 추가).
  - 값이 0인 항목은 범례에서 자동 제외 처리.

### 2.2 학반 관리 및 학생 데이터 고도화
- **학년별 맞춤 옵션**: 발달 단계에 따른 기초진로희망 및 특이사항 드롭다운 차별화.
- **기타(직접입력)**: 드롭다운에서 항목 선택 시 즉시 텍스트 입력 모드로 전환되는 지능형 UI 구현.
- **데이터 시각화**: 모든 드롭다운 항목에 고유 색상(variant)을 적용하여 가독성 증대.
- **편집 권한 확장**: 학생 현황 페이지에서도 관리자가 기초진로희망을 직접 수정할 수 있도록 개방.

### 2.3 시스템 안정성 및 성능 최적화
- **Undo/Redo (Ctrl+Z/Y)**: 로컬 상태와 서버 동기화를 완벽하게 처리하는 다단계 실행 취소/다시 실행 엔진 구현.
- **렌더링 최적화**: `useMemo`, `useCallback`, `memo`를 전방위적으로 적용하여 대규모 데이터 환경에서의 `mousedown`, `keydown` 지연 현상 해결.
- **하이드레이션 수정**: SSR 환경에서 Radix UI ID 불일치로 인한 오류 해결.

## 3. 향후 권장 사항
- **데이터 클렌징**: '진로미결정' 인원이 많은 학급에 대한 별도 관리 알림 기능 검토.
- **리포트 연동**: 현재 대시보드의 학과별/반별 통계 데이터를 PDF 리포트로 출력하는 기능 추가 가능.
