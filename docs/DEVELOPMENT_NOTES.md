# 대공커리어싱크 개발 가이드 및 참고사항 (Reference)

이 문서는 대공커리어싱크 프로젝트의 핵심 비즈니스 로직과 개발 규칙을 담고 있습니다. 새로운 기능을 추가하거나 수정할 때 반드시 참조하세요.

---

## 📅 1. 학사학년도 및 학년 공식 (Academic Logic)

대공커리어싱크는 '졸업연도'가 아닌 **'현재 학사학년도'**를 기준으로 학생의 학년을 계산합니다.

- **기본 공식**: `Grade = 4 - (GraduationYear - AcademicYear)`
- **조회 공식 (데이터 패칭 시)**: `GraduationYear = AcademicYear + (4 - Grade)`
- **동작 예시 (2026학년도 기준)**:
  - 3학년 조회: `2026 + (4 - 3) = 2027년` 졸업 예정자
  - 2학년 조회: `2026 + (4 - 2) = 2028년` 졸업 예정자
  - 1학년 조회: `2026 + (4 - 1) = 2029년` 졸업 예정자

> **주의**: 시스템 설정의 `baseYear`가 바뀌면 전체 학생의 '현재 학년'이 자동으로 한 학년씩 올라가거나 내려가게 설계되어 있습니다.

## 🗄️ 2. 데이터 소스 및 타입 규칙 (Data & Types)

- **절대 금지**: 로컬 JSON 파일에 설정값을 저장하지 마세요. 모든 마스터 데이터는 **Supabase DB**를 사용합니다.
- **타입 분리**: 클라이언트 컴포넌트용 타입 및 상수는 `src/lib/types.ts`에 정의하세요. `src/lib/data.ts`는 서버 전용 코드입니다.
- **학과 정렬 표준 (`MAJOR_SORT_ORDER`)**: 자동화기계과 ~ 섬유소재과까지의 순서를 엄격히 준수하세요.
- **핵심 데이터 필드 매핑**:
  - `career_aspiration`: 진로희망 (취업, 진학, 제외인정자)
  - `special_notes`: 희망 기업유형 (대/공기업, 공무원 등)
  - `career_course`: 희망 진로코스 (청솔반, 도제반, 군특성화 등)
  - `personal_remarks`: **비고(특이사항)** - 학급 관리 및 학생 관리 페이지 공통 연동 필드

## 🎨 3. UI/UX 컨벤션

### 3.1 필터 및 헤더 (Layout)
- **반응형 제목**: 창 너비가 좁아질 때 제목이 깨지지 않도록 `xl (1280px)` 기준으로 레이아웃을 전환하며, 제목과 설명에는 `whitespace-nowrap`을 적용합니다.
- **필터 배치**: 세로 배치 시 항상 제목이 상단, 필터가 하단에 위치하도록 순서를 고정합니다.
- **지능형 초기값**: 관리자/비담임은 3학년, 담임 교사는 본인 담당 학년(`assigned_grade`)이 초기값으로 자동 선택됩니다.

### 3.2 차트 (Charts)
- **공통 규격**: 높이는 **`h-[300px]`**, 막대 굵기는 **`barSize={20}`**, 범례는 **하단 중앙** 배치를 원칙으로 합니다.
- **그리드 배치**: 데스크톱 환경에서는 모든 차트를 **2열 구조(`lg:grid-cols-2`)**로 배치하여 빈 공간 없이 연속적으로 나열합니다.
- **원형/도넛 차트 표준**: 시작 지점 **9시 방향 (`startAngle={180}`)**, 회전 방향 **시계 방향 (`endAngle={-180}`)**, 지시선(`labelLine`) 사용.
- **색상 표준**:
    - 취업(예): `#10b981` (Emerald-500) / 미취업(아니오): `#ef4444` (Red-500)
    - 제외인정자: `#94a3b8` (Slate-400) / 진학: `#6366f1` (Indigo-500)
    - 자격증 분포: **Sky Blue 그라데이션** (0개: `#cbd5e1` ~ 6개이상: `#0369a1`)
- **데이터 부재 처리**: 분석 데이터가 0인 경우 시각적인 플레이스홀더와 안내 메시지를 표시합니다.

### 3.3 테이블 (Tables)
- **성능 최적화**: 대량 데이터 표시 시 반드시 고정 높이와 `useCallback`/`React.memo`를 적용하여 불필요한 재렌더링을 방지합니다.
- **동적 옵션**: `options` 속성에 함수를 전달하여 진로희망 값에 따라 드롭다운 목록을 동적으로 변경합니다.
- **지능형 직접 입력**: '기타(직접입력)' 선택 시 즉시 `Input` 모드로 전환되며, 값 미입력 시 `null`로 저장됩니다.
- **Undo/Redo 지원**: 테이블 내 수정 사항은 `Ctrl + Z`, `Ctrl + Y`를 통해 최대 20단계까지 복구 가능합니다.

### 3.4 하이드레이션 및 모바일 표준
- **하이드레이션 방지**: `useIsMobile` 사용 시 서버/클라이언트 불일치를 막기 위해 컴포넌트 마운트(`mounted` 상태) 이후에 동적 UI를 렌더링해야 합니다.
- **카드 UI 표준**: 헤더는 `{학과} {반}반 {번호}번` 형식을 사용하며, 주요 정보는 **3열 2행 (총 6개)** 구성을 원칙으로 합니다.
- **데이터 초기화**: '선택 취소(비우기)'는 클라이언트에서 `CLEARED` 상수를 전달하며, 서버에서 이를 `null`로 변환하여 DB에 반영합니다.

## 📊 4. 통계 및 용어 규칙 (Statistics & Terminology)

### 4.1 용어 체계
- **진로희망**: 대분류 (취업, 진학, 제외인정자).
- **희망 기업유형**: 취업 희망 시 타겟 기업 성격. (제외인정자/진학 시 선택 안함)
- **희망 진로코스**: 학교 내 세부 교육 과정 및 진학 형태.
- **비고(특이사항)**: 학생의 개인별/행정적 특이사항을 기록하는 공식 명칭.

### 4.2 취업률 산정 공식
- `취업률 = 취업자 / (전체 인원 - 제외인정자) * 100`

### 4.3 학년별 핵심 지표
- **1학년**: 진로 희망 분포, 자격증 취득 시작 현황, 학생 특성(진로 코스) 분석.
- **2학년**: 진로 구체화(아우스빌둥, 도제 등), 전공 역량(자격증), 병역 희망 분석.
- **3학년**: 실전 취업률, 기업 규모별 분포, 현장실습 참여 및 채용 전환 분석.
