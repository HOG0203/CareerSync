# 대공커리어싱크 개발 가이드 및 참고사항 (Reference)

이 문서는 대공커리어싱크 프로젝트의 핵심 비즈니스 로직과 개발 규칙을 담고 있습니다. 새로운 기능을 추가하거나 수정할 때 반드시 참조하세요.

## 📅 1. 학사학년도 및 학년 공식 (Academic Logic)

대공커리어싱크는 '졸업연도'가 아닌 **'현재 학사학년도'**를 기준으로 학생의 학년을 계산합니다.

- **기본 공식**: `Grade = 4 - (GraduationYear - AcademicYear)`
- **조회 공식 (데이터 패칭 시)**: `GraduationYear = AcademicYear + (4 - Grade)`
- **동작 예시 (2026학년도 기준)**:
  - 3학년 조회: `2026 + (4 - 3) = 2027년` 졸업 예정자
  - 2학년 조회: `2026 + (4 - 2) = 2028년` 졸업 예정자
  - 1학년 조회: `2026 + (4 - 1) = 2029년` 졸업 예정자

- **절대 금지**: 로컬 JSON 파일에 설정값을 저장하지 마세요. 모든 마스터 데이터는 **Supabase DB**를 사용합니다.

- **학과 정렬 표준 (`MAJOR_SORT_ORDER`)**: 자동화기계과 ~ 섬유소재과까지의 지정된 순서를 엄격히 준수하세요.
- **필드 매핑 규칙**:
  - `career_aspiration`: 진로희망 (취업, 진학, 제외인정자)
  - `special_notes`: 희망 기업유형 (대/공기업, 공무원 등)
  - `career_course`: 희망 진로코스 (청솔반, 도제반, 군특성화, 전문대 등)

## 🎨 3. UI/UX 컨벤션

### 3.1 필터 및 헤더 (Layout)
- **반응형 제목**: 창 너비가 좁아질 때 제목이 깨지지 않도록 `xl (1280px)` 기준으로 레이아웃을 전환하며, 제목에는 `whitespace-nowrap`을 적용합니다.
- **필터 배치**: 세로 배치 시 항상 제목이 상단, 필터가 하단에 위치하도록 `flex-col` 순서를 고정합니다.
- **배타적 학년 필터**: 대시보드 필터에서 1, 2, 3학년 중 하나를 반드시 선택해야 합니다.

### 3.2 차트 (Charts)
- **공통 규격**: 높이는 **`h-[300px]`**, 막대 굵기는 **`barSize={20}`**, 범례는 **하단 중앙** 배치를 원칙으로 합니다.
- **그리드 배치**: 데스크톱 환경에서는 모든 차트를 **2열 구조(`lg:grid-cols-2`)**로 배치하여 빈 공간 없이 연속적으로 나열합니다.
- **도넛 차트 표준**: 시작 지점 **9시 방향 (`startAngle={180}`)**, 회전 방향 **시계 방향 (`endAngle={-180}`)**.
- **색상 테마**:
    - 취업 관련: Emerald/Blue 계열
    - 진학 관련: Indigo/Violet 계열
    - 제외인정자 관련: Teal/Lime/Yellow 계열
    - 미설정/기타: Slate/Grey 계열

### 3.3 테이블 (Tables)
- **동적 옵션 (Dependency Select)**: `options` 속성에 함수를 전달하여 '진로희망' 값에 따라 '기업유형' 및 '진로코스' 목록을 동적으로 변경합니다.
- **지능형 직접 입력**: '기타(직접입력)' 선택 시 즉시 `Input` 모드로 전환됩니다.
- **Undo/Redo**: 최대 20단계까지 복구 가능한 기능을 유지합니다.

## 📊 4. 통계 및 용어 규칙 (Statistics & Terminology)

### 4.1 용어 체계
- **진로희망**: 대분류 (취업, 진학, 제외인정자).
- **희망 기업유형**: 취업 희망 시 타겟으로 하는 기업의 성격.
- **희망 진로코스**: 학교 내 세부 교육 과정 (도제, 청솔반, 군특성화 등).

### 4.2 취업률 산정 공식
- `취업률 = 취업자 / (전체 인원 - 제외인정자) * 100`
