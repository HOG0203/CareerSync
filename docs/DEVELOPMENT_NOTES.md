# CareerSync 개발 가이드 및 참고사항 (Reference)

이 문서는 CareerSync 프로젝트의 핵심 비즈니스 로직과 개발 규칙을 담고 있습니다. 새로운 기능을 추가하거나 수정할 때 반드시 참조하세요.

---

## 📅 1. 학사학년도 및 학년 공식 (Academic Logic)

CareerSync는 '졸업연도'가 아닌 **'현재 학사학년도'**를 기준으로 학생의 학년을 계산합니다.

- **기본 공식**: `Grade = 4 - (GraduationYear - AcademicYear)`
- **조회 공식 (데이터 패칭 시)**: `GraduationYear = AcademicYear + (4 - Grade)`
- **동작 예시 (2026학년도 기준)**:
  - 3학년 조회: `2026 + (4 - 3) = 2027년` 졸업 예정자
  - 2학년 조회: `2026 + (4 - 2) = 2028년` 졸업 예정자
  - 1학년 조회: `2026 + (4 - 1) = 2029년` 졸업 예정자

> **주의**: 시스템 설정의 `baseYear`가 바뀌면 전체 학생의 '현재 학년'이 자동으로 한 학년씩 올라가거나 내려가게 설계되어 있습니다.

## 🗄️ 2. 데이터 소스 및 타입 규칙 (Data & Types)

- **절대 금지**: 로컬 JSON 파일(`src/lib/settings.json` 등)에 설정값을 저장하거나 읽지 마세요.
- **표준**: 모든 시스템 설정 및 마스터 데이터는 **Supabase DB**를 사용합니다.
- **타입 분리**: 클라이언트 컴포넌트에서 사용하는 타입 및 상수는 반드시 `src/lib/types.ts`에 정의하세요. `src/lib/data.ts`는 서버 전용 코드를 포함하므로 클라이언트에서 임포트하면 빌드 에러가 발생합니다.
- **학과 정렬 표준 (`MAJOR_SORT_ORDER`)**: 
  1. 자동화기계과, 2. 친환경자동차과, 3. 자동차기계과, 4. 스마트공간과, 5. 건설과, 6. 스마트공간건축과, 7. 스마트전기과, 8. 전기과, 9. 바이오화학과, 10. 스마트융합섬유과, 11. 섬유소재과 순서를 엄격히 준수하세요.

## 🎨 3. UI/UX 컨벤션

### 3.1 필터 (Filters)
- 필터 컴포넌트(`DashboardFilters`) 사용 시 반드시 `baseYear`를 전달하세요.
- **상태 필터 기준**: 대시보드의 상태 필터는 `employment_status`가 아닌 **`business_type` (취업여부)**를 참조해야 통계와 일관성을 유지할 수 있습니다.
- **디자인 표준**: 
    - `slate-50` 배경과 아이콘 조합 유지.
    - 콤팩트한 레이아웃을 위해 항목 내부 여백은 **`px-1`**, 아이콘 간격은 **`gap-1`**을 사용합니다.
    - 레이아웃 시프트 방지를 위해 트리거 너비를 고정(`w-[130px]` 등) 하세요.

### 3.2 차트 (Charts)
- **공통 규격**: 높이는 **`h-[300px]`**, 막대 굵기는 **`barSize={20}`**, 범례는 **하단 중앙(`bottom-center`)** 배치를 원칙으로 합니다.
- **색상 표준**:
    - 취업(예): `#10b981` (Emerald-500)
    - 미취업(아니오): `#ef4444` (Red-500)
    - 제외인정자: `#94a3b8` (Slate-400)
    - 자격증 분포: 6개 이상(`#1e3a8a`)에서 0개(`#cbd5e1`)까지 푸른색 그라데이션 사용.
- **레이아웃**: 모바일 **1열**, 데스크탑 **2열** 그리드(`lg:grid-cols-2`)를 사용합니다.

### 3.3 모바일 (Mobile)
- 모바일 전용 모달은 `MobileDetailModal`을 사용하며, 모든 입력 필드에는 **지우기(X)** 기능을 포함해야 합니다.
- 드롭다운 선택 해제 시에는 `CLEARED` 상수를 처리하는 로직을 포함하세요.

### 3.4 테이블 (Tables)
- **성능 최적화 (가상화)**: 대량의 데이터를 표시하는 테이블 페이지는 반드시 **고정된 높이(`calc`)**를 가져야 합니다. `h-full` 사용 시 가상화가 작동하지 않아 브라우저가 매우 느려질 수 있습니다.
    - 권장 높이: `h-[calc(100vh-135px)]` (모바일), `h-[calc(100vh-95px)]` (데스크탑)
- **UI 통합**: 테이블 상단의 `CardHeader` 타이틀은 가급적 제거하고, 동적인 정보(조회 인원 등)는 페이지 부제목 영역에 통합하여 수직 공간을 확보합니다.
- **자동 너비**: 콘텐츠 길이에 맞게 셀이 늘어날 수 있도록 `table-auto`와 `min-width` 조합을 사용합니다.
- **정렬**: 학생 목록은 항상 **학과 순서(`MAJOR_SORT_ORDER`) -> 반 -> 번호** 순으로 정렬되어야 합니다.
- **데이터 정제**: 불필요한 'X' 표시는 화면상에서 빈칸으로 렌더링하여 가독성을 높입니다.

## 📊 4. 통계 산출 규칙 (Statistics)

### 4.1 취업률 산정 공식
- **취업자**: `business_type === '예'`
- **제외 대상**: `business_type === '제외인정자'` (전체 모수에서 차감)
- **공식**: `취업률 = 취업자 / (전체 인원 - 제외인정자) * 100`
- **주의**: 대시보드 요약 카드 및 모든 취업 관련 차트는 이 공식을 엄격히 따라야 합니다.

## 🚀 5. 배포 및 환경 설정

- **빌드 전 검증**: 배포 전 반드시 `npm run typecheck`를 실행하여 타입 에러가 없는지 확인하세요.
- **Server Actions**: 데이터 수정 시 반드시 `revalidatePath`를 호출하여 실시간 갱신을 보장하세요.
- **RLS (Row Level Security)**: 새로운 테이블 생성 시 관리자(admin)와 교사(teacher)의 접근 권한을 Supabase 대시보드에서 반드시 설정해야 합니다.
