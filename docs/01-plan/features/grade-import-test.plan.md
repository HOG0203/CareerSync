# PDCA Plan: 학생 성적 엑셀 일괄 입력 테스트 시스템

## 1. 개요 (Overview)
- **목표**: 엑셀 파일(`전기3-1.xlsx`)을 활용하여 학생들의 성적 데이터를 시스템에 일괄적으로 정확하게 입력하는 기능을 개발하고, 이를 테스트할 수 있는 전용 페이지를 구축함.
- **주요 기능**:
  - **엑셀 파일 업로드 및 파싱**: `xlsx` 라이브러리를 사용하여 로컬 엑셀 데이터를 JSON으로 변환.
  - **데이터 매칭 로직**: 학과(스마트전기과), 학년(3학년), 반(1반) 정보를 기반으로 이름과 번호를 대조하여 DB 레코드와 매칭.
  - **일괄 데이터 업데이트**: 매칭된 학생 레코드에 성적 정보를 서버 액션을 통해 일괄 업데이트.
  - **테스트용 내비게이션**: 개발 및 검증 단계에서 쉽게 접근할 수 있도록 사이드바 및 모바일 내비게이션에 메뉴 추가.

## 2. 분석 및 설계 (Analysis & Design)

### 2.1 데이터 모델링 및 환경 분석
- **대상 데이터**: `전기3-1.xlsx` (2026학년도 스마트전기과 3학년 1반 데이터).
- **데이터베이스 연동**: 
  - 현재 `student_employments` 테이블에는 성적 필드가 부재함.
  - 테스트 목적으로 신규 컬럼(예: `academic_score`)을 추가하거나 별도의 성적 관리 테이블 구성을 검토.
- **매칭 식별자**: `student_name` + `student_number` (동명이인 방지를 위해 조합하여 식별).

### 2.2 UI/UX 설계
- **테스트 페이지 경로**: `/admin/grade-import-test`
- **주요 UI 요소**:
  - 파일 선택(File Input) 버튼.
  - 엑셀 데이터 미리보기 테이블.
  - 매칭 성공/실패 여부 표시.
  - '최종 적용하기' 버튼 (서버 반영).

## 3. 단계별 추진 계획 (Step-by-Step Plan)

### Phase 1: 파일 구조 분석 및 DB 준비
- [ ] `전기3-1.xlsx` 파일의 컬럼 구성 확인 (국어, 영어, 수학 등 개별 과목인지 혹은 평균 성적인지).
- [ ] `student_employments` 테이블에 성적 데이터를 저장할 컬럼(`academic_grade` 등) 추가 SQL 작성 및 실행.

### Phase 2: 서버 액션 및 유틸리티 구현
- [ ] Excel 데이터를 파싱하여 DB 스키마에 맞게 변환하는 유틸리티 함수 작성.
- [ ] 학생 정보를 조회하고 일괄 업데이트하는 서버 액션 구현.

### Phase 3: 테스트 UI 및 내비게이션 통합
- [ ] `/app/(dashboard)/admin/grade-import-test/page.tsx` 생성 및 UI 구현.
- [ ] 사이드바(`src/components/dashboard/nav.tsx`)와 모바일 탭(`src/components/dashboard/mobile-bottom-tab.tsx`)에 "성적 입력(T)" 메뉴 추가.

### Phase 4: 검증 및 동기화
- [ ] 실제 `전기3-1.xlsx` 파일을 업로드하여 3학년 1반 학생들의 데이터가 정확히 매칭되는지 확인.
- [ ] 데이터베이스 반영 결과 확인 후 실제 페이지 적용 여부 결정.

## 4. 기대 결과 (Expected Outcome)
- 수동 입력 시 발생할 수 있는 휴먼 에러를 방지하고 대량의 데이터를 신속하게 업데이트 가능.
- 향후 전 학년/전 학과 성적 입력 시스템으로 확장하기 위한 기술적 기반 마련.
