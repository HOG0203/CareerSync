# PDCA Plan: 관리자 통합 학생 등록 및 진급 시스템 구축

## 1. 개요 (Overview)
- **목표**: 관리자가 전 학년 학생 데이터를 통합 관리하고, 학기 초/말 대규모 학적 변동(등록, 진급)을 안전하고 빠르게 처리할 수 있는 전용 관리 허브 구축.
- **주요 기능**:
  - **전 학년 통합 조회**: 1, 2, 3학년 전체 데이터 접근 권한.
  - **대량 등록 허브**: 신입생 및 전입생 일괄 임포트 최적화.
  - **전교생 진급 도구**: 반별 이동이 아닌 학년 단위 대규모 진급 처리 지원.
  - **연도 관리 연동**: 시스템 기준 연도(`baseYear`) 변경과 진급 로직 통합.

## 2. 분석 및 설계 (Analysis & Design)

### 2.1 페이지 구성 (`/admin/students`)
- **상단**: 학년/학과/반 통합 검색 필터 및 전체 통계 배지.
- **중단**: 전 학년 통합 스프레드시트 (편집 가능).
- **우측/하단 플로팅**: 일괄 작업 메뉴 (가져오기, 진급 모달, 삭제).

### 2.2 진급(Promotion) 고도화 로직
- **매칭 방식 추가**: 
  1. **수동 지정**: 현재처럼 모달에서 입력.
  2. **CSV 매칭**: [이름, 새 학과, 새 반, 새 번호]가 담긴 파일을 업로드하여 기존 UUID와 매칭 후 일괄 업데이트.

## 3. 단계별 추진 계획 (Step-by-Step Plan)

### Phase 1: 관리자 전용 페이지 기반 구축
- [ ] `src/app/(dashboard)/admin/students/page.tsx` 생성 및 권한 체크.
- [ ] 전 학년 데이터를 불러오기 위한 전용 쿼리 함수 구현.

### Phase 2: 통합 필터 및 테이블 구성
- [ ] `DashboardFilters`를 확장하여 1~3학년 전체 필터링 지원.
- [ ] 대용량 데이터를 처리할 수 있는 최적화된 테이블 렌더링.

### Phase 3: 전교생 단위 진급 시스템 구현
- [ ] 학년 전체를 선택하여 다음 단계로 넘기는 진급 워크플로우 개발.
- [ ] 진급 전 데이터 백업(Snapshot) 기능 연동.

### Phase 4: 사이드바 및 네비게이션 연동
- [ ] 관리자 메뉴에 '학생/진급 관리' 항목 추가.

## 4. 기대 결과 (Expected Outcome)
- 수백 명의 학생 이동 작업을 실시간으로 모니터링하며 안전하게 완료.
- 관리자의 데이터 정비 업무 시간 80% 이상 단축.
- 학적 이력의 누락 없는 영구 보존 체계 완성.
