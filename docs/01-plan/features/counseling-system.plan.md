# PDCA Plan: 누적 상담 관리 시스템 고도화

## 1. 개요 (Overview)
- **목표**: 학생별 3년간의 상담 내역을 누적으로 관리하고, 편리하게 기록/조회할 수 있는 모달 기반의 상담 시스템 구축.
- **주요 기능**:
  - **상담 히스토리 보존**: 진급과 상관없이 학생 고유 ID에 귀속된 모든 상담 내역 유지.
  - **상세보기 모달**: 테이블 우측의 버튼을 통해 과거~현재의 모든 상담 기록을 확인.
  - **누적 기록**: 새로운 상담 내용을 실시간으로 추가하고 관리자와 공유.
  - **UI 개선**: 상담 관련 컬럼을 가장 우측으로 배치하여 접근성 강화.

## 2. 분석 및 설계 (Analysis & Design)

### 2.1 데이터 모델링
- **신규 테이블**: `student_counseling_logs`
  - `id`: UUID (PK)
  - `student_id`: UUID (FK, `student_employments.id`)
  - `content`: TEXT (상담 내용)
  - `author_name`: TEXT (작성 교사 성명)
  - `created_at`: TIMESTAMPTZ (기록 일시)

### 2.2 UI/UX 설계
- **학반 관리 테이블**: 
  - `counseling_log` 컬럼을 마지막 순서로 이동.
  - 셀 내에 텍스트 대신 '상세보기' 아이콘/버튼 배치.
- **상담 관리 모달**:
  - 상단: 학생 기본 정보 (학과, 학년, 성명).
  - 중단: 타임라인 형태의 상담 기록 리스트.
  - 하단: 신규 상담 기록 입력창 및 저장 버튼.

## 3. 단계별 추진 계획 (Step-by-Step Plan)

### Phase 1: 데이터베이스 고도화
- [ ] `student_counseling_logs` 테이블 생성 및 RLS 정책 설정.
- [ ] 기존 `counseling_log` 필드의 데이터를 신규 테이블로 이전하는 마이그레이션 스크립트 작성.

### Phase 2: 상담 관리 서버 액션 구현
- [ ] 상담 내역 호출 함수 (`getHistoryAction`).
- [ ] 상담 내용 저장 함수 (`saveLogAction`).

### Phase 3: 상담 상세보기 모달 UI 개발
- [ ] Shadcn UI `Dialog`를 활용한 상담 히스토리 뷰어 구현.
- [ ] 상담 일자별 소팅 및 작성자 표시 로직 적용.

### Phase 4: 학반 관리 테이블 연동
- [ ] `ClassTable` 컬럼 순서 조정 및 상세보기 버튼 연동.
- [ ] 최근 상담 내용의 요약본(최근 1건) 노출 기능 추가.

### Phase 5: 최종 검증
- [ ] 관리자/담임교사 계정별 상담 기록 권한 확인.
- [ ] 다년도 데이터가 올바르게 합산되어 표시되는지 테스트.

## 4. 기대 결과 (Expected Outcome)
- 학생 한 명에 대한 깊이 있는 이해와 연속성 있는 진로 지도 가능.
- 복잡한 엑셀 관리 없이 시스템 내에서 모든 상담 이력을 통합 관리.
- 교사 간(학년 전환 시) 인수인계 효율성 극대화.
