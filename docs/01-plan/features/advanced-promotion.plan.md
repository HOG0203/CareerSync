# PDCA Plan: 진급 시스템 고도화 (학급 정보 일괄 매핑)

## 1. 개요 (Overview)
- **목표**: 진급 처리 시 졸업연도 변경뿐만 아니라, 새로운 학과, 반, 번호를 일괄 또는 개별적으로 지정할 수 있는 고도화된 진급 시스템 구축.
- **주요 기능**:
  - **진급 설정 모달**: 선택된 학생들의 진급 후 정보를 미리 설정하는 UI.
  - **일괄 학급 지정**: 선택 인원 전체의 반을 한 번에 변경하는 기능.
  - **번호 자동/수동 할당**: 진급 후 번호를 목록에서 직접 수정.
  - **데이터 연속성**: 기존 상담일지, 자격증 데이터를 신규 학년 데이터로 완벽 인계.

## 2. 분석 및 설계 (Analysis & Design)

### 2.1 데이터 처리 로직
- **입력 데이터**: `[{ id, next_major, next_class, next_number, next_year }]` 형태의 배열.
- **ID 처리**: 고유 UUID 기반 업데이트로 기존 연관 데이터(상담일지 등) 자동 유지.
- **식별자 갱신**: `student_id` (학과_반_번호_성명_연도) 필드를 새로운 정보로 재조합하여 업데이트.

### 2.2 UI/UX 설계
- **진급 버튼 트리거**: 테이블 행 선택 시 나타나는 '진급 처리' 버튼 클릭.
- **설정 화면**:
  - 좌측: 현재 정보 (학년, 반, 번호, 성명).
  - 우측: 진급 후 정보 입력칸 (Default: 현재 정보 유지 + 연도-1).
  - 상단: "전체 [ ] 반으로 지정" 퀵 액션.

## 3. 단계별 추진 계획 (Step-by-Step Plan)

### Phase 1: 서버 액션 확장
- [ ] `promoteStudents` 함수가 상세 업데이트 객체 배열을 받도록 수정.
- [ ] 트랜잭션 처리를 통한 일괄 업데이트 로직 안정화.

### Phase 2: 진급 설정 모달 개발
- [ ] `PromotionModal` 컴포넌트 신규 생성.
- [ ] 선택된 학생 리스트 렌더링 및 입력 폼(Input, Select) 구성.
- [ ] 일괄 반 변경 기능 로직 구현.

### Phase 3: 테이블 연동
- [ ] `StandardSpreadsheetTable`의 `onPromote` 콜백이 모달을 띄우도록 수정.
- [ ] 처리 결과 피드백(성공/실패 토스트) 강화.

### Phase 4: 최종 테스트
- [ ] 1학년 전체 진급 시 데이터 정합성 확인.
- [ ] 반/번호 변경 후 상담일지 이력 조회 테스트.

## 4. 기대 결과 (Expected Outcome)
- 학년 교체기 대규모 학생 이동 작업을 단 몇 분 만에 정확하게 처리.
- 학생 지도 데이터의 단절 없는 3년 관리 체계 완성.
- 교직원의 수동 데이터 수정 작업 90% 이상 절감.
