# PDCA Plan: 학년별 분리 관리 및 진급 시스템 구축

## 1. 개요 (Overview)
- **목표**: 학년별 특성에 맞는 관리 페이지 분리 및 진급 시 데이터 연속성 확보.
- **주요 전략**:
  - **3학년 통합 관리**: '학생 취업 및 현장실습 현황' 페이지에서 전체 3학년의 취업 데이터 집계.
  - **학반별 개별 관리**: '학반관리' 페이지에서 1, 2, 3학년 담임교사가 개별 상담 및 기초 데이터를 관리.
  - **데이터 연동**: 학반관리에서 입력한 데이터가 3학년의 경우 통합 관리 페이지에 실시간 반영.
  - **진급 시스템**: 학년 전환 시 상담 이력과 기초 데이터를 유지하며 소속 정보만 일괄 변경.

## 2. 분석 및 설계 (Analysis & Design)

### 2.1 페이지별 역할 정의
- **학생 취업 현황 페이지**: 관리자용. 전체 3학년의 취업/실습 현황 통계 및 리포트 중심.
- **학반관리 페이지**: 담임교사용. 1~3학년 학생의 개별 상담, 자격증, 기초 진로 데이터 입력 중심.

### 2.2 진급(Promotion) 로직
- 1학년 → 2학년, 2학년 → 3학년 진급 시 `graduation_year`를 변경하고 상담 데이터 보존.

## 3. 단계별 추진 계획 (Step-by-Step Plan)

### Phase 1: 학반관리 페이지 기능 확장
- [ ] '학반관리' 페이지에 학년(1, 2, 3) 선택 필터 추가.
- [ ] 담임교사가 상담 내용을 입력할 수 있는 전용 컬럼(상담일지, 기초희망 등) 추가.

### Phase 2: 3학년 통합 페이지 최적화
- [ ] 3학년 데이터만 필터링하여 출력되도록 고정.
- [ ] 학반관리에서 입력된 상담 기초 데이터가 취업 현황에 연동되도록 구성.

### Phase 3: 진급 시스템 구현
- [ ] 학년 전환 시 소속 정보만 업데이트하는 '진급 처리' 기능 개발.
